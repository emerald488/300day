# 300 Day Challenge

**Представьте:** каждый день вы становитесь сильнее. Сегодня — одно повторение, завтра — два, через неделю — семь. На 100-й день вы выполните сотню отжиманий, приседаний, подтягиваний. А к финалу, на 300-й день, ваше тело преодолеет барьер в 300 повторений каждого упражнения и 15 минут планки. Это не просто цифры — это трансформация. От простого движения к абсолютной силе и выносливости. Маленькие шаги каждый день складываются в невероятный результат: железная дисциплина, несгибаемая воля и тело, способное на то, о чём вы раньше только мечтали.

## Начать челлендж

**[Открыть приложение](https://emerald488.github.io/300day/challenge-tracker.html)** — начните свой путь к трансформации прямо сейчас!

Минималистичный PWA-трекер для ежедневного прогресса в пяти упражнениях: отжимания, приседания, подтягивания, подъемы по лестнице и планка.

## Возможности

### 🏋️ Тренировки
- **Прогрессивная нагрузка**: день 1 = 1 повторение, день 100 = 100 повторений, день 300 = 300 повторений
- **5 упражнений**: отжимания, приседания, подтягивания, лестница (пролеты), планка (секунды)
- **Умный таймер планки**: отображение с миллисекундами, пауза/возобновление, ручное добавление секунд
- **Быстрые кнопки**: +1/+5/+10 повторений (для лестницы дополнительно +16)
- **Визуальный прогресс**: цветные прогресс-бары, автоматическая подсветка выполненных упражнений
- **Защита от дублирования**: один раз в день, блокировка повторного завершения

### 📊 Статистика и история
- **Детальная статистика**: общие показатели по всем упражнениям
- **Средние значения**: автоматический расчет среднего в день
- **История тренировок**: последние 30 дней с полной детализацией
- **Счетчик стрика**: отслеживание серии дней подряд

### 💾 Данные и безопасность
- **Локальное хранение**: все данные в `localStorage` браузера (без серверов)
- **Экспорт/импорт**: резервное копирование в JSON-формате
- **Автосохранение**: мгновенное сохранение после каждого действия

### 📱 Интеграции
- **Telegram-уведомления**: подключение личного бота для получения детальных отчетов о каждой тренировке
- **Yandex Metrika**: аналитика использования (анонимная)

### 🎨 Интерфейс
- **Stories-онбординг**: приветственные экраны при первом запуске
- **Нижняя навигация**: быстрый доступ к главной, статистике, истории и настройкам
- **Темная тема**: удобный интерфейс для глаз
- **Адаптивный дизайн**: отлично работает на телефонах, планшетах и десктопе

### 🔌 PWA (Progressive Web App)
- **Офлайн-режим**: работа без интернета
- **Установка на устройство**: как нативное приложение
- **Автообновление**: новые версии подгружаются автоматически
- **Кроссплатформенность**: iOS, Android, Windows, macOS, Linux

## Быстрый старт

### Веб-версия
1. **Откройте приложение**: [https://emerald488.github.io/300day/challenge-tracker.html](https://emerald488.github.io/300day/challenge-tracker.html)
2. **Пройдите онбординг**: посмотрите приветственные stories (показываются один раз в день)
3. **Выполняйте упражнения**: используйте кнопки +1/+5/+10 для добавления повторений
4. **Запустите таймер планки**: нажмите "Старт" или добавляйте секунды вручную
5. **Завершите день**: кнопка "День выполнен" сохранит прогресс и перейдет к следующему дню

### Установка как PWA

#### На iPhone/iPad (Safari):
1. Откройте приложение в Safari
2. Нажмите кнопку "Поделиться" (квадрат со стрелкой вверх)
3. Выберите "На экран «Домой»"
4. Нажмите "Добавить"

#### На Android (Chrome):
1. Откройте приложение в Chrome
2. Нажмите меню (три точки)
3. Выберите "Установить приложение" или "Добавить на главный экран"
4. Подтвердите установку

#### На Desktop (Chrome/Edge):
1. Откройте приложение
2. В адресной строке нажмите иконку установки (⊕ или компьютер)
3. Нажмите "Установить"

### Резервное копирование
- **Экспорт**: Настройки → Экспорт данных (сохраняет JSON-файл)
- **Импорт**: Настройки → Импорт данных (восстанавливает из файла)
- **Рекомендация**: делайте бэкап раз в неделю!

## Структура
- `challenge-tracker.html` - основной интерфейс, логика работы с прогрессом и PWA-обновлениями.
- `service-worker.js` - кеширование ключевых файлов, обновление через `skipWaiting`.
- `manifest.json` - метаданные приложения и иконки (`icon-192.png`, `icon-512.png`).

## FAQ (Частые вопросы)

**Q: Что будет, если я пропущу день?**
A: Серия дней (стрик) обнулится, но общая статистика сохранится. Можно продолжить с текущего дня.

**Q: Можно ли изменить количество повторений за прошлые дни?**
A: Нет, история доступна только для просмотра. Это помогает поддерживать честность челленджа.

**Q: Как перенести прогресс на другое устройство?**
A: Используйте функцию экспорта данных в настройках, затем импортируйте файл на новом устройстве.

**Q: Что делать, если случайно очистил кэш браузера?**
A: Если у вас есть экспортированный файл, просто импортируйте его. Без бэкапа данные восстановить невозможно.

**Q: Можно ли делать больше повторений, чем цель?**
A: Да! Приложение позволяет превышать цель. Перевыполнение будет видно в истории.

**Q: Работает ли приложение без интернета?**
A: Да, после первой загрузки PWA работает полностью офлайн. Интернет нужен только для Telegram-уведомлений.

**Q: Сколько места занимают данные?**
A: Очень мало — около 10-50 КБ даже после 300 дней. Все хранится в текстовом формате JSON.

## Ограничения и заметки

- **Данные локальны**: хранятся только в браузере. Очистка кэша/данных сбросит прогресс.
- **Один браузер**: данные не синхронизируются между разными браузерами автоматически.
- **Безопасность**: не вводите токен Telegram на общедоступных устройствах.
- **Кодировка**: все файлы в UTF-8.
- **Приватность**: данные не отправляются на сервера (кроме Telegram, если настроен).

### Обновление версии
При внесении изменений не забудьте обновить:
1. Версию в `manifest.json` (поле `version`)
2. `CACHE_NAME` в `service-worker.js`
3. Параметры `?v=X` в `challenge-tracker.html`

### Структура кода
- **app.js**: модульная структура с комментариями по разделам
- **Функции именованы понятно**: `completeDay()`, `addReps()`, `updateUI()`
- **Глобальное состояние**: объект `data` с вложенной структурой
- **События**: все обработчики через `onclick` в HTML для простоты
