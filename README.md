# 300 Day Challenge

Минималистичный PWA-трекер для ежедневного прогресса в пяти упражнениях: отжимания, приседания, подтягивания, подъемы по лестнице и планка.

## Возможности
- Локальное хранение: весь прогресс, история и стрики лежат в `localStorage` браузера.
- Ежедневный рост целей: номер дня = цель по повторениям, для планки цель = день * 3 секунды.
- Управление подходами: быстрые кнопки +1/+5/+10 (лестница еще +16), сброс упражнения, автоматический таймер для планки.
- Завершение дня: проверка выполненности всех целей, сохранение записи в истории (до 30 дней), увеличение целей и счетчика стрика.
- Статистика и история: суммарные показатели по каждому упражнению, видимость по переключателям, короткая хронология.
- Импорт/экспорт: резервная копия прогресса в JSON и восстановление из файла.
- Telegram-уведомления: подключение личного бота для получения уведомлений о завершении каждого дня с детальной статистикой.
- PWA-режим: `manifest.json`, сервис-воркер (`service-worker.js`) с кэшированием `challenge-tracker.html`; можно установить на устройство и работать офлайн.

## Быстрый старт
1. Откройте `challenge-tracker.html` в современном браузере (Chrome/Edge/Firefox). Для установки как PWA позвольте браузеру добавить страницу на главный экран.
2. Добавляйте повторения кнопками в карточке упражнения. Для планки используйте "Старт"/"Стоп" или добавляйте секунды вручную.
3. Нажмите "День выполнен", чтобы зафиксировать результат, обновить цели на следующий день и сохранить запись в истории.
4. Экспортируйте данные перед сменой устройства и импортируйте файл при первом запуске.

## Структура
- `challenge-tracker.html` - основной интерфейс, логика работы с прогрессом и PWA-обновлениями.
- `service-worker.js` - кеширование ключевых файлов, обновление через `skipWaiting`.
- `manifest.json` - метаданные приложения и иконки (`icon-192.png`, `icon-512.png`).

## Ограничения и заметки
- Данные хранятся локально; очистка кэша/данных браузера или другой браузер сбросит прогресс.
- Сервис-воркер кеширует ограниченный набор файлов. При изменениях обновляйте версию кеша (`CACHE_NAME`).
- Все тексты в интерфейсе на русском; убедитесь, что файл открыт в кодировке UTF-8.
